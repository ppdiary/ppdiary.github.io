## 原來是捲軸在搞鬼

後來我才明白，當頁面內容很長、需要向下滾動時，右側會出現一條大約 15px 寬的捲軸（Scrollbar），擠壓了中央空間。而內容少少的頁面不需要捲軸，中央自然不會被擠壓，這就是標題位置不一致的原因。

## 解決方法很簡單  
要讓捲軸固定保持空間，Gemini 教我兩種方式，一個快速直接，另一個是比較漂亮的作法。

### 第一種：基礎作法 [^1]
在 CSS 中加入 `html {
    overflow-y: scroll;
}`，這會強制瀏覽器在所有頁面都預留捲軸空間，無論內容長短都有捲軸，中央空間固定了下來，標題也就不再跑來跑去。

不過你可能會想：「短頁面不需要捲軸吧，這樣看起來醜醜的。」
我也這麼覺得，所以最後採用了更漂亮的作法。

### 第二種：漂亮的作法 [^2]
後來我改用 `html {
    scrollbar-gutter: stable; 
}` 這段指令，和第一種一樣也會永遠保留捲軸的寬度空間，但只在需要時才顯示，短頁面不會出現捲軸，視覺上乾淨許多。

就這樣，一行 CSS 輕鬆搞定，改好之後超開心的。
